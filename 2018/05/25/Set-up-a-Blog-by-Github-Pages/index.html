Set up a Blog by Github Pages

发表于 2018-05-25

用GitHub搭博客

按照Hexo+GitPage搭建个人博客一步步来，一步一个坑啊…

换个git bash的名字

以前装过git，是老的用户名和密码，想换成新的，问题是，不管卸了重装还是怎么样，一打开git bash都是老的用户名和密码

查了很多，有人说 control panel - user accounts - manage windows credentials，删掉github 的credential就行了，但是，删了之后还不行，删了之后重装了还不行。难道要重启…?试试看

重启了，但是还是不行。生气

改了电脑名字和Microsoft的账户名，电脑名字倒是改了，但是用户名还是没改

设置了一个新的本地用户Miao 123456，然后再重新登录Microsoft的账号，这下改了，但是问题是… 名字是Miao@Miao，为什么不是Miaomiao@Miao

重装了git bash 还是这样… 算了

右击gitbash 的图标，选择properties，原来是这样

然后把–cd-to-home删掉，把start in改成我们自己设置的文件夹E:\Git Bash，重新打开后就好了

安装blog

一步一步按照blog上的指示，安装好hexo啦

然后，部署到GitHub pages上

在GitHub上 new repository，名字必须是username.github.io，initialize this repository with a README 打勾

打开yml文件，写入type, repository, branch的时候，注意不要加多余的空格，就写好一行，回车就行了，不然会出现很多问题

终于！终于弄好了，打开了主页，留下了激动的泪水

等等…

不！并没有…

问题

localhost里面是很顺利的显示了主页，但是问题在于，miaomiao1005.github.io上面，还是原来的那样空白…

解决

全部重装了一遍，就好了，不知道为什么…

  • 可能是因为这次文件夹名字是Blog/Hexo，而不是上次随便起的名字…？
  • 这次也重新设置了一遍SSH
  • 这次安装一步一步完全按照blog上写的哦

主页的配置

打开blog文件夹的themes目录，然后 git clone [想要clone的主题GitHub地址]

然后把blog文件夹里面的_config.yml文件用notepad打开，修改参数为 theme: hexo-theme-next 。注意这里，很多地方说是要改成 theme: next，我试了试，不行，只能是hexo-theme-next（估计是因为theme里面的文件夹叫hexo-theme-next而不是next吧）

回到blog文件夹，打开gitbash，输入 hexo g和hexo s，然后就可以在http://localhost:4000/看见主页了，这时主题是默认的

想要换主题的话，就把blog文件夹里的themes/hexo-theme-next里面的_config.yml文件里面的主题改成Gemini

发表文章

可以在blog文件夹里，git bash，然后hexo n “标题”，这样就可以在blog文件夹的source/_posts里面生成一个md文档

或者可以直接在这个_post文件夹里创建一个md，写完后用hexo g和hexo d推送到服务器上
