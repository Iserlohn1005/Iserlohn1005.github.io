又遇到了新的问题，在 sudo apt install git的时候，就Could not get lock /var/lib/dpkg/lock.. 查了一下是因为还有一个线程在用apt-get。可以重启电脑，或者：ps -A | grep apt-get ，搜索所有运行着的线程，找到第一列是apt的线程，记下第二列的线程号，然后sudo kill 线程号就行了。但是我这里输入ps -A | grep apt-get没有任何反应…

哦我知道了 ps -A是用来看线程的，而grep apt-get是用来筛选有apt-get的内容，而我用的是apt而不是apt-get… 输入 ps -A | grep apt 就能得到结果了。

非常好，终于kill了这些线程，然后安装上了git.. clone之后，在home里面就能找到binwalk文件，然后在这个文件夹里打开终端，sudo python setup.py install 安装。这一步可算是装好啦…

然后，在IDA 的程序里，右击DATA XREF，选择 list cross references to，就能看到引用的地方（或者双击也可以）

临时：XREF

IDA XREF

交叉引用分为 CODE XREF(代码交叉引用)和 DATA XREF(数据交叉引用)。

代码交叉引用

表示一条指令将控制权转交给另一条指令，这叫流flow。流分为普通流，调用流，跳转流。普通流就是默认的一条条往下执行，调用流是↓p(procedure) ：

①显示了两次，表示这个函数被调用了2次

跳转流，每个无条件分支指令和条件分支指令将分配到一个跳转流

虚线表示相邻的两条指令之间并不存在普通流（也就是00401048后没有跟着顺序执行的指令）

#####数据交叉引用

用于跟踪二进制文件访问数据的方式。与虚拟地址的字节有关，与栈变量无关。分为三种，读取交叉引用，写入交叉引用，偏移量交叉引用。

读取交叉引用，read cross reference, ↑r , read_it在_main+E处、_main+25被读取

写入交叉引用，指出了修改变量内容的位置，↑w ，write_it在_main+1B、_main+2E处被写入

偏移量交叉引用，表示引用的是某个位置的地址（而不是内容），后缀↑o（看做是Offset）。 ref_it在_main+4处被引用
